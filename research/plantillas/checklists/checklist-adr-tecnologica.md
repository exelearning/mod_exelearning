# Checklist: ADR tecnológica

Antes de marcar un ADR como `Aceptada`:

- [ ] Contexto y problema enunciados con precisión
- [ ] Mínimo 2 opciones evaluadas
- [ ] Cada opción cita ≥1 evidencia (REPO/FTE/AN/EXP)
- [ ] Si compara estándares: existe matriz cuantificada (no sólo cualitativa)
- [ ] Riesgos enumerados con su `RIE-NNN` (o nota explícita de ausencia)
- [ ] Plan de validación concreto (experimento, tests, métricas)
- [ ] Consecuencias positivas y negativas listadas
- [ ] Cambios disparados en otros ADRs/tareas anotados
- [ ] `herramienta_ia` registrada si se generó con IA
